## 1. Базовый шаблон (`base.html`)

Базовый каркас всех страниц приложения.

### Технологии

- **Bootstrap 5** — CSS-фреймворк (CDN)
- **Font Awesome 6** — иконки (CDN)
- **Кастомные CSS** — `style.css`, `blog-layout.css`
- **JavaScript** — `theme-toggle.js` (синхронно в `<head>` для предотвращения «вспышки» темы)

### Структура

```
<!DOCTYPE html>
<html data-bs-theme="dark|light">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Flask Blog - {% block title %}{% endblock %}</title>
    <!-- Bootstrap 5 CSS -->
    <!-- Font Awesome -->
    <!-- style.css -->
    <!-- blog-layout.css -->
    <!-- theme-toggle.js (синхронно, до рендеринга body) -->
  </head>
  <body>
    <!-- Навбар -->
    <nav class="navbar navbar-expand-lg">
      <!-- Бренд: Flask Blog -->
      <!-- Навигация -->
      <!-- Правая часть: переключатель темы + кнопки -->
    </nav>

    <!-- Основной контент -->
    <main class="container mt-4">
      <!-- Flash-сообщения -->
      {% with messages = get_flashed_messages(with_categories=true) %}
        ...
      {% endwith %}
      {% block content %}{% endblock %}
    </main>

    <!-- Футер -->
    <footer class="footer mt-auto py-3">
      Flask Blog © 2026
    </footer>

    <!-- Bootstrap 5 JS -->
    <!-- navbar.js -->
    <!-- auth-cookies.js -->
    <!-- login-discriminator-update.js -->
  </body>
</html>
```

### Навбар

Адаптивная навигация с условной логикой:

| Состояние | Элементы |
|---|---|
| **Гость** | «Войти», «Регистрация» |
| **Пользователь** | «Новый пост», дропдаун с `login_full`, «Выход» |
| **Админ** | Всё как у пользователя + «Админ панель» |

Справа: переключатель темы (checkbox `themeSwitch`).

### Flash-сообщения

Используются алерты Bootstrap 5 с цветовой категоризацией:
- `success` → `alert-success`
- `error` / `danger` → `alert-danger`
- `warning` → `alert-warning`
- `info` → `alert-info`

### Футер

Sticky footer через flexbox (`d-flex flex-column min-vh-100` на `<body>`).

## 2. Страницы аутентификации

### login.html — Вход

**Путь:** `/auth/login`  
**Наследует:** `base.html`

**Содержание:**
- Карточка с формой входа
- Поле «Логин» (input `type="text"`, `maxlength="50"`, `autocomplete="username"`)
- Поле «Пароль» (input `type="password"`, `autocomplete="current-password"`)
- Чекбокс «Запомнить меня на 30 дней» (`remember_me`)
- Кнопка «Войти»
- Блок ошибки (`{% if error %}`)
- Подсказка: «Система автоматически определит ваш аккаунт по дискриминатору»

**Скрипты:**
- `auth-cookies.js` — сохранение/восстановление последнего логина
- `login-discriminator-update.js` — обновление дискриминатора в cookie

### register.html — Регистрация

**Путь:** `/auth/register`  
**Наследует:** `base.html`

**Содержание:**
- Карточка с формой регистрации
- Поле «Логин» (`maxlength="50"`, `autocomplete="username"`)
- Поле «Пароль» (`autocomplete="new-password"`)
- Поле «Подтверждение пароля» (`autocomplete="new-password"`)
- Кнопка «Зарегистрироваться»
- Блок ошибки (`{% if error %}`)
- Ссылка «Уже есть аккаунт? Войти»

### select_account.html — Выбор аккаунта

**Путь:** `/auth/select-account`  
**Наследует:** `base.html`

**Содержание:**
- Отображается, когда найдено несколько аккаунтов с одинаковым логином
- Радиокнопки с `login#discriminator` для каждого найденного пользователя
- Поле пароля
- Кнопка «Войти»
- Информация: «Найдено несколько аккаунтов с логином X. Выберите нужный.»

## 3. Страницы блога

### index.html — Список постов

**Путь:** `/`  
**Наследует:** `base.html`

**Содержание:**
- Сетка постов (карточки Bootstrap)
- Каждая карточка: заголовок (ссылка на пост), автор, дата, превью текста (`post.body_preview(200)`)
- Для авторизованных: кнопка «Новый пост»
- Навигация «предыдущая/следующая» (пагинация)
- Пустое состояние: «Постов пока нет» с кнопкой создания

### create_post.html — Создание/редактирование

**Путь:** `/post/create` или `/post/<id>/edit`  
**Наследует:** `base.html`

**Содержание:**
- Заголовок «Новый пост» или «Редактирование поста»
- Поле «Заголовок» (input `type="text"`)
- Поле «Содержание» (textarea, многострочный ввод)
- Кнопка «Опубликовать» / «Сохранить изменения»
- При редактировании: кнопка «Удалить пост» (POST-форма)

### post_detail.html — Просмотр поста

**Путь:** `/post/<id>`  
**Наследует:** `base.html`

**Содержание:**
- Заголовок поста
- Автор и дата
- Полный текст поста (с `nl2br` фильтром)
- Для автора или админа: кнопки «Редактировать» и «Удалить»
- **Секция комментариев:**
  - Список комментариев (автор, дата, текст)
  - Для автора комментария или админа: кнопка «Удалить»
  - Для авторизованных: форма добавления комментария (textarea + кнопка)
- Пустое состояние комментариев: «Комментариев пока нет»

## 6. JavaScript

### theme-toggle.js

**Переключатель тёмной/светлой темы.**

Функциональность:
- Загружается **синхронно в `<head>`** для предотвращения «вспышки» (flash of wrong theme)
- Сохраняет состояние в `localStorage` (ключ `theme`)
- Синхронизирует с атрибутом `data-bs-theme` на `<html>`
- Поддерживает cookie `theme` для серверной стороны
- Обработчик изменения на checkbox `#themeSwitch`

### navbar.js

**Логика навигации.**

- Подсветка активного пункта меню
- Адаптивное поведение при изменении размера окна

### auth-cookies.js

**Управление cookies аутентификации.**

- Сохранение последнего логина в cookie `last_login`
- Восстановление логина при загрузке страницы
- Управление `last_full_login` (логин с дискриминатором)

### login-discriminator-update.js

**Обновление дискриминатора в cookie.**

- После выбора аккаунта на `select_account.html` обновляет cookie `last_full_login`
- Используется для правильного автозаполнения при следующем входе
