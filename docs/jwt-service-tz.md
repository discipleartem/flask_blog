# ТЗ: JWT Service для Flask блога

## 🎯 Обзор

Самописный JWT сервис для авторизации пользователей в Flask блоге. Использует HMAC-SHA256 для подписи токенов и HttpOnly cookies для хранения.

## 📋 Функциональные требования

### Основные методы:
1. **генерироватьТокен**
   - Создает JWT токен по ID пользователя
   - Параметры: user_id (int), remember_me (bool, опционально)
   - Возвращает: строка токена
   - Срок действия: 24ч стандартно, 30дней с remember_me

2. **проверитьТокен**
   - Проверяет валидность токена
   - Параметры: token (str)
   - Возвращает: payload словарь при успехе, None при ошибке
   - Проверяет срок действия и подпись

3. **создатьПодпись**
   - Создает HMAC-SHA256 подпись для payload
   - Параметры: payload (str)
   - Возвращает: строка подписи
   - Использует SECRET_KEY из конфигурации

4. **создатьPayload**
   - Создает JSON payload с временными метками
   - Параметры: user_id (int), exp_hours (int)
   - Возвращает: закодированная строка payload
   - Содержит поля user_id, exp, iat

## 🔧 Технические требования

### Безопасность:
- **Алгоритм**: HS256 жестко закодирован в классе
- **Секрет**: Использовать config.SECRET_KEY
- **Кодирование**: Base64 URL-safe, без символа "=" в конце
- **Время**: Unix timestamp в секундах

### Структура токена:
- **Header**: Не хранится (алгоритм в коде)
- **Payload**: `{"user_id": int, "exp": int, "iat": int}`
- **Signature**: HMAC-SHA256(secret_key, payload)
- **Формат**: `payload.signature`

### Поля payload:
- `user_id`: ID пользователя
- `exp`: Unix timestamp истечения срока
- `iat`: Unix timestamp создания

## 🛡️ Проверки безопасности

### При верификации:
1. **Формат токена**: Проверка наличия точки-разделителя
2. **Payload декодирование**: Base64 URL-safe декодирование
3. **Срок действия**: `exp > current_time`
4. **Подпись**: Сравнение через `hmac.compare_digest`
5. **Структура**: Наличие обязательных полей

### Обработка ошибок:
- Неверный формат токена → None
- Истекший токен → None
- Неверная подпись → None
- Ошибка декодирования → None

## 🔗 Интеграция

### Зависимости:
- Стандартные библиотеки: `hmac`, `hashlib`, `base64`, `json`, `time`
- Конфигурация: `config.SECRET_KEY`

### Использование в Flask:
- Установка токена в `response.set_cookie()`
- Чтение токена из `request.cookies.get()`
- Параметры cookie: HttpOnly, Secure, SameSite=Lax

### Параметры cookie:
- **Имя**: `auth_token`
- **HttpOnly**: True
- **Secure**: True (в production)
- **SameSite**: Lax
- **Max-age**: соответствует сроку действия токена

## 📝 Пример использования

### Генерация токена:
- Создать экземпляр JWTService
- Вызвать generate_token с user_id=123, remember_me=False
- Установить полученный токен в cookie с параметрами:
  - Имя: auth_token
  - HttpOnly: True
  - Max-age: 86400 секунд (24 часа)

### Верификация токена:
- Получить токен из request.cookies
- Вызвать verify_token с токеном
- При успехе извлечь user_id из payload

## ✅ Критерии приемки

1. Токен генерируется с правильной структурой
2. Токен верифицируется в течение срока действия
3. Токен отвергается после истечения срока
4. Токен отвергается при неверной подписи
5. Поддерживается опция "запомнить меня" (30 дней)
6. Используются только стандартные библиотеки
7. Корректная обработка ошибок без исключений
