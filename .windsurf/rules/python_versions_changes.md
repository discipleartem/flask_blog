# Изменения в версиях Python 3.11 - 3.14

## Python 3.11 (Октябрь 2022)

### 🚀 Основные улучшения

#### Производительность
- **Ускорение на 10-60%** по сравнению с Python 3.10
- Средний прирост производительности **1.25x** на стандартном наборе тестов
- **Специализирующий адаптивный интерпретатор** (PEP 659) - анализирует код во время выполнения и заменяет общие байткоды специализированными версиями
- **Быстрый старт** - запуск интерпретатора на 10-15% быстрее
- **Оптимизация фреймов** - вызовы функций Python стали значительно быстрее

#### Улучшенные сообщения об ошибках
- **Точное указание на ошибку** (PEP 657) - интерпретатор указывает на точное выражение, вызвавшее ошибку, а не только на строку
- Улучшенная трассировка для вложенных словарей и множественных вызовов функций

### 📋 Новые возможности языка

#### Обработка исключений
- **Exception Groups** (PEP 654) - возможность поднимать и обрабатывать несколько несвязанных исключений одновременно
- Новые типы `ExceptionGroup` и `BaseExceptionGroup`
- Новый синтаксис `except*` для обработки групп исключений
- Метод `add_note()` для добавления контекстной информации к исключениям (PEP 678)

#### Типизация
- **Variadic Generics** - поддержка обобщенных типов с произвольным количеством параметров
- **typing.Self** - аннотация для методов класса, возвращающих self
- **typing.LiteralString** - указание строковых литералов
- **Улучшения dataclasses** - новые возможности для классов данных

### 📚 Стандартная библиотека

#### asyncio
- **Task Groups** - новая концепция для организации асинхронных задач
- Улучшенная обработка ошибок с использованием Exception Groups

#### enum
- **StrEnum** - новый базовый класс для строковых перечислений

#### Другие улучшения
- **Нативная поддержка TOML** - встроенная работа с конфигурационными файлами
- **Атомные группы в регулярных выражениях** - `(?>...)` и possessive квантификаторы `*+`, `++`, `?+`, `{m,n}+`

## Python 3.12 (Октябрь 2023)

### 🚀 Основные улучшения

#### F-строки
- **Синтаксическая формализация f-строк** (PEP 701) - f-строки теперь обрабатываются PEG-парсером
- Возможность переиспользовать кавычки внутри f-строк
- Поддержка многострочных выражений
- Поддержка комментариев и обратных слешей внутри выражений
- Более точные сообщения об ошибках

#### Производительность
- **Прирост производительности ~5%** в целом
- **Оптимизация comprehensions** (PEP 709) - списковые, словарные и множественные включения теперь встроенные (inline), ускорение до 2x
- **Immortal Objects** (PEP 683) - объекты с бесконечным счетчиком ссылок для лучшей оптимизации
- **Уменьшение размера объектов** - базовый размер объекта уменьшен с 208 до 96 байт

### 📋 Новые возможности языка

#### Типизация - новый синтаксис
- **Параметры типов** (PEP 695) - новый компактный синтаксис для generic-типов
  ```python
  def max[T](args: Iterable[T]) -> T: ...
  class Stack[T]: ...
  ```
- **Ключевое слово `type`** - для создания type aliases
- **TypedDict для kwargs** - типизация **kwargs с разными типами
- **`@override` декоратор** - явное указание переопределения методов

#### Отладка и профилирование
- **Новый API для отладки/профилирования** (PEP 669) - низкоуровневые хуки с минимальными накладными расходами
- **Поддержка Linux perf profiler** - отображение имен Python-функций в трассировках

### 🔧 Другие возможности

#### Субинтерпретаторы
- **Per-interpreter GIL** (PEP 684) - каждый субинтерпретатор может иметь свой GIL
- Позволяет использовать преимущества многоядерных процессоров
- Пока доступно только через C-API

#### Улучшенные сообщения об ошибках
- Предложения для модулей стандартной библиотеки при NameError
- Умные подсказки при ошибках импорта
- Предложения при неправильном синтаксисе импорта

#### Удалено
- Модуль **distutils** удален из стандартной библиотеки (PEP 632)
- Удалены устаревшие модули: `asynchat`, `asyncore`, `imp`
- **setuptools** больше не устанавливается по умолчанию в venv

## Python 3.13 (Октябрь 2024)

### 🚀 Основные улучшения

#### Новый интерактивный интерпретатор (REPL)
- Основан на коде из проекта PyPy
- **Многострочное редактирование** с сохранением истории
- **Цветная подсветка синтаксиса** в реальном времени
- **Автодополнение** для модулей в import
- **Цветные трассировки** по умолчанию
- **F1** - интерактивная помощь (pydoc)
- **F2** - просмотр истории
- **F3** - режим вставки для больших блоков кода

#### Экспериментальная поддержка свободной потоковости (Free-threaded)
- **PEP 703** - экспериментальный режим без GIL
- Потоки могут работать более параллельно
- Требуется специальная сборка (обычно `python3.13t`)
- Возможность использовать все доступные ядра CPU

#### JIT-компилятор
- **Экспериментальный JIT** (PEP 744) - Just-In-Time компиляция
- Закладывает основу для значительных улучшений производительности
- Использует технологию copy-and-patch
- Новая зависимость от LLVM на этапе сборки

### 📋 Новые возможности языка

#### Типизация
- **Значения по умолчанию для параметров типов**
- **typing.TypeIs** - новая аннотация для сужения типов
- **typing.ReadOnly** - маркировка read-only элементов в TypedDict
- **Аннотации для обозначения устаревшего кода**

#### locals()
- **PEP 667** - определенная семантика при изменении возвращаемого mapping
- Отладчики теперь могут более надежно обновлять локальные переменные

### 🔧 Другие возможности

#### Улучшенные сообщения об ошибках
- Продолжение традиции улучшения сообщений
- Более точные подсказки при типичных ошибках
- Цветной вывод в unittest, argparse, json, calendar

#### Стандартная библиотека
- **dbm.sqlite3** - новый backend для dbm (по умолчанию)
- **warnings.deprecated()** - новый декоратор для отметки устаревшего кода
- **asyncio introspection** - инструменты для отладки асинхронного кода

#### Платформы
- **macOS** - минимальная версия изменена с 10.9 на 10.13
- **WASI** - теперь платформа Tier 2
- **Android** - теперь платформа Tier 3
- **Emscripten** - больше не официально поддерживается

#### Удалено (PEP 594)
Множество устаревших модулей: `aifc`, `audioop`, `chunk`, `cgi`, `cgitb`, `crypt`, `imghdr`, `mailcap`, `msilib`, `nis`, `nntplib`, `ossaudiodev`, `pipes`, `sndhdr`, `spwd`, `sunau`, `telnetlib`, `uu`, `xdrlib`, `lib2to3`

#### Виртуальные окружения
- Виртуальные окружения теперь автоматически игнорируются Git

## Python 3.14 (Октябрь 2025)

### 🚀 Основные улучшения

#### Free-threaded Python - официальная поддержка
- **PEP 779** - Free-threaded сборка теперь официально поддерживается
- Больше не экспериментальная функция
- Истинный параллелизм для потоков в Python
- Каждый интерпретатор имеет собственный GIL

#### Отложенное вычисление аннотаций
- **PEP 649** - аннотации больше не вычисляются немедленно
- Аннотации сохраняются в специальных annotate-функциях
- Вычисляются только при необходимости
- Улучшение производительности и удобства использования
- Модуль `annotationlib` для работы с аннотациями

#### Template Strings (T-строки)
- **PEP 750** - новый тип строк для пользовательской обработки
- Синтаксис похож на f-строки, но с префиксом `t`
- Создают объект `Template` вместо немедленного вычисления
- Позволяют безопасно обрабатывать пользовательский ввод
- Разделение статических сегментов и интерполяций

### 📋 Новые возможности языка

#### Субинтерпретаторы в стандартной библиотеке
- **PEP 734** - модуль `concurrent.interpreters`
- Множественные изолированные интерпретаторы в одном процессе
- Каждый с собственным GIL
- Поддержка Communicating Sequential Processes (CSP)
- `InterpreterPoolExecutor` в concurrent.futures

#### Улучшения синтаксиса
- **PEP 758** - `except` и `except*` теперь могут опускать скобки
- Упрощенный синтаксис обработки исключений

### 🔧 REPL и CLI

#### Подсветка синтаксиса
- **Синтаксическая раскраска в REPL** в реальном времени
- Экспериментальная поддержка тем
- Цветной вывод в стандартных модулях:
  - `argparse` - цветные справочные сообщения
  - `calendar` - подсветка текущего дня
  - `json` - цветной вывод JSON
  - `unittest` - цветной вывод для тестов

#### Автодополнение
- Автодополнение имен модулей в import-выражениях

### 🐛 Отладка

#### Удаленная отладка
- **PEP 768** - интерфейс для внешних отладчиков с нулевыми накладными расходами
- `pdb` поддерживает удаленное подключение к процессу
- Возможность подключить отладчик к уже запущенному процессу
- Инструмент командной строки для инспекции запущенных процессов

#### asyncio introspection
- Инструменты для понимания и отладки асинхронного кода
- CLI для инспекции запущенных Python-процессов с async задачами

### 📚 Стандартная библиотека

#### Сжатие
- **PEP 784** - новый модуль `compression.zstd`
- Поддержка алгоритма сжатия Zstandard

### ⚡ Производительность

#### Новый tail-call интерпретатор
- Использует tail calls между маленькими C-функциями
- Предоставляет **3-5% улучшение производительности** на pyperformance
- Работает с Clang 19+ на x86-64 и AArch64
- Опциональная функция (требует PGO для лучших результатов)

#### JIT-компилятор
- Экспериментальный JIT доступен в Windows и macOS
- По умолчанию выключен
- Может быть включен через `PYTHON_JIT=1`

### 🔧 Другие улучшения

#### Улучшенные сообщения об ошибках
- Подсказки для опечаток в ключевых словах Python
- Более точные предложения при синтаксических ошибках

#### Платформы
- **Emscripten** - теперь официально поддерживаемая платформа (Tier 3)
- **Android** - официальные бинарные релизы
- **Windows** - новый менеджер установки Python

#### Изменения в безопасности
- **PEP 761** - Python 3.14+ больше не предоставляет PGP-подписи
- Рекомендуется использовать Sigstore для верификации

## Рекомендации по обновлению

### Python 3.11
- Отличный выбор для проектов, которым нужен баланс стабильности и производительности
- Значительный прирост скорости
- Статус: **Security-fixes only** (до октября 2027)

### Python 3.12
- Хороший выбор для проектов с активным использованием f-строк и типизации
- Улучшенная производительность и новый синтаксис типов
- Статус: **Security-fixes only** (до октября 2028)

### Python 3.13
- Рекомендуется для новых проектов
- Отличный новый REPL для разработки
- Экспериментальная поддержка free-threading для будущего
- Статус: **Stable** (поддержка до октября 2029)

### Python 3.14
- Новейшая стабильная версия
- Официальная поддержка free-threading для многопоточных приложений
- Template strings для безопасной обработки строк
- Субинтерпретаторы для продвинутых сценариев параллелизма
- Статус: **Stable** (поддержка до октября 2030)
