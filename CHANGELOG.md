# Changelog

Все значимые изменения проекта будут задокументированы в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/)
и этот проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

## [0.3.0] - 2026-01-27

### Added
- 🔐 **Уникальная система аутентификации** с дискриминаторами в стиле Discord (Username#1234)
- 📝 **CRUD операции для постов блога** с правами доступа
- 💬 **Система комментариев** с валидацией и управлением
- 🛡️ **CSRF защита** с HMAC-SHA256 подписью токенов
- 🚀 **CI/CD пайплайн** с GitHub Actions
- 🧪 **Комплексная система тестирования** с покрытием >80%
- 🏗️ **Service Layer архитектура** для разделения логики
- 📊 **Отчёты о покрытии** и безопасности в CI/CD

### Security
- 🔒 PBKDF2-HMAC-SHA256 хэширование паролей с 100k итераций
- 🛡️ Защита от XSS атак через markupsafe.Markup
- 🔐 Безопасные cookie с флагами Secure/HttpOnly/SameSite=Strict
- 🚫 Автоматическая валидация CSRF токенов во всех формах
- 👤 Система прав доступа на основе авторизации

### Fixed
- ✅ **Исправлены все ошибки mypy** (32 → 0 ошибок)
- 🐛 Устранены критические уязвимости безопасности (bandit)
- 🔧 Оптимизирована работа с базой данных SQLite
- 📝 Улучшена обработка форм и валидация данных
- 🎨 Исправлены проблемы с отображением в шаблонах

### Technical
- 📦 Обновлены зависимости до совместимых версий
- 🏛️ Реализован Application Factory паттерн
- 🗃️ Чистый SQL без ORM для максимального контроля
- 📋 Добавлены маркеры для категоризации тестов
- 🔍 Интегрированы инструменты статического анализа

### Documentation
- 📖 Подробное описание архитектуры аутентификации
- 🛠️ Инструкции по установке и разработке
- 📊 Описание CI/CD пайплайна
- 🎯 Примеры использования API endpoints

## Версионирование

- **Major (X.0.0)** - Обратные несовместимые изменения
- **Minor (0.X.0)** - Новая функциональность с обратной совместимостью  
- **Patch (0.0.X)** - Исправления ошибок с обратной совместимостью

## Типы изменений

- `Added` - Новая функциональность
- `Changed` - Изменения в существующей функциональности
- `Deprecated` - Функциональность, которая будет удалена в будущих версиях
- `Removed` - Удалённая функциональность
- `Fixed` - Исправления ошибок
- `Security` - Улучшения безопасности
